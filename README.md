## What Spectator Is

Spectator is a browser-based, single-file HTML app for exploring digital brush collections with a more hands-on visual workflow. It can open Procreate `.brush` and `.brushset` files, along with Photoshop `.abr` files, then show the contents in a preview area and an information panel so you can review brushes before exporting or reorganizing them.

Since everything runs on the client side, the files remain in the browser while you work. That makes Spectator useful as a lightweight brush inspector, a fast packager for chosen assets, or an exporter for previews and extracted image resources without needing a heavier desktop setup.

## How to Use It

1. Open Spectator in your browser.
2. Drag and drop a Procreate or Photoshop brush file into the app, or select a file manually.
3. Review the preview area and brush information panel.
4. Export previews in the layout and format you need.

6. Split a brushset into individual brushes or package selected brushes into a new brushset.

Typical workflow:

- Inspect the brushes visually
- Export previews or image assets
- Rebuild brush collections as needed

## Common Questions

### Does Spectator upload my brush files anywhere?
No external upload step is part of the workflow described here. The app works in the browser and reads files locally during use.

### Can I use it on GitHub Pages?
Yes. The project structure is suitable for browser hosting, and the provided download link points to the GitHub Pages placeholder.

### What formats are supported?
Spectator works with Procreate `.brush` and `.brushset` files, plus Photoshop `.abr` files.

### How do I update it?
Replace your hosted copy or pull the latest repository changes, then reopen the app in your browser.

### What if a file does not open correctly?
Check that the file matches one of the supported brush formats and try a newer browser session. If the issue persists, verify the source file and reload the app.

### Where are settings stored?
Configuration is handled in the browser session and interface, so there is no separate desktop settings directory mentioned in the project metadata.
